# 07 — นโยบายลิขสิทธิ์ + ข้อความพร้อมใช้ในแอป

> ⚠️ เอกสารนี้เป็นแนวปฏิบัติของทีม **ไม่ใช่คำปรึกษาทางกฎหมาย** — ก่อนเปิดฟีเจอร์ "เผยแพร่สาธารณะ" (เฟส 5) ควรให้ที่ปรึกษากฎหมายตรวจอีกครั้ง

## 1. ข้อเท็จจริงที่ทีมต้องเข้าใจตรงกัน

- เพลงหนึ่งเพลงมีลิขสิทธิ์ซ้อนกันหลายชั้น: **ดนตรีกรรม** (ทำนอง+เนื้อร้อง), **สิ่งบันทึกเสียง** (มาสเตอร์), และ**การแสดง** — ตาม พ.ร.บ.ลิขสิทธิ์ พ.ศ. 2537
- **คอร์ดล้วน ๆ (ทางเดินคอร์ด)** โดยทั่วไปถือเป็นข้อเท็จจริง/ไอเดียทางดนตรี — ความเสี่ยงต่ำ แต่**เนื้อเพลง**คืองานอันมีลิขสิทธิ์ชัดเจน การทำซ้ำ/เผยแพร่เนื้อเพลงเต็มโดยไม่ได้รับอนุญาต = ละเมิด
- การแกะเพลง **เพื่อศึกษา/ใช้ส่วนตัว ไม่แสวงกำไร** เข้าข่ายข้อยกเว้น (มาตรา 32) ได้ — แต่การ**เผยแพร่ต่อสาธารณะ**ไม่เข้า
- การดาวน์โหลดเสียงจาก YouTube ขัด **YouTube Terms of Service** (แม้จะไม่ใช่ประเด็นลิขสิทธิ์โดยตรง) — เป็นความเสี่ยงระดับ platform

## 2. หลักการออกแบบผลิตภัณฑ์ (สรุปเป็น 4 ข้อ)

1. **Private by default** — ผลการแกะอยู่บนเครื่องผู้ใช้ (IndexedDB) ไม่ขึ้น server ยกเว้นผู้ใช้สั่ง sync/เผยแพร่เอง
2. **ผู้ใช้เป็นผู้ยืนยันสิทธิ์** — แสดงคำเตือนก่อนแกะครั้งแรก + ก่อนเผยแพร่ทุกครั้ง (ข้อความ §4)
3. **ไม่เก็บไฟล์เสียงถาวร** — ไฟล์ต้นทาง/สเต็มลบภายใน 24 ชม. (docs/01 §6) เก็บเฉพาะผลลัพธ์ (คอร์ด/เนื้อ/แท็บ) ของผู้ใช้
4. **มีช่องทาง takedown** — สารบัญสาธารณะ (เฟส 5) ต้องมีปุ่ม report + อีเมลติดต่อ + ลบภายใน 7 วันเมื่อเจ้าของสิทธิ์ร้องขอ

## 3. กติกาใน repo (สำหรับทีมพัฒนาเอง)

- เพลงตัวอย่าง/เดโม/ชุดทดสอบที่ commit ลง repo ต้องเป็น **เพลงแต่งเองหรือ public domain เท่านั้น**
- ห้าม commit ไฟล์เสียงที่มีลิขสิทธิ์ / เนื้อเพลงจริงของศิลปิน ลงใน repo, test fixture, หรือ screenshot ในเอกสาร
- ชุดทดสอบ 20 เพลง (docs/02 §11) เก็บเป็น **ลิงก์อ้างอิง + ไฟล์เฉลยคอร์ด** ในเครื่องทีมเท่านั้น ไม่ push

## 4. ข้อความพร้อมใช้ในแอป (ก๊อปไปใช้ได้เลย)

### 4.1 Modal ก่อนเริ่มแกะครั้งแรก (บังคับกดยอมรับ 1 ครั้ง)

> **ก่อนเริ่มใช้ AquaChord**
>
> 🎵 AquaChord ช่วยแกะคอร์ด เนื้อเพลง และแท็บด้วย AI เพื่อ**การเรียนรู้และฝึกซ้อมส่วนตัว**
>
> - ผลการแกะจะถูกเก็บไว้**ในเครื่องของคุณเท่านั้น** จนกว่าคุณจะเลือกแชร์เอง
> - เพลงส่วนใหญ่มีลิขสิทธิ์ — การนำเนื้อเพลง/คอร์ดไปเผยแพร่ต่อสาธารณะหรือใช้เชิงพาณิชย์ ต้องได้รับอนุญาตจากเจ้าของสิทธิ์
> - โปรดใช้ไฟล์เสียงที่คุณมีสิทธิ์ใช้งาน การดาวน์โหลดจากบางแพลตฟอร์มอาจขัดข้อกำหนดการใช้งานของแพลตฟอร์มนั้น
>
> [ ยอมรับและเริ่มใช้งาน ]   [ อ่านนโยบายฉบับเต็ม ]

### 4.2 ข้อความใต้ฟอร์มวาง URL (แสดงตลอด — ตัวเล็ก)

> ใช้เพื่อการศึกษาส่วนตัวเท่านั้น · คุณต้องมีสิทธิ์ในไฟล์เสียงที่นำมาแกะ

### 4.3 Modal ก่อนกด "เผยแพร่สู่สารบัญกลาง" (เฟส 5 — ทุกครั้ง)

> **ยืนยันก่อนเผยแพร่**
>
> เพลงนี้จะปรากฏในสารบัญสาธารณะพร้อมชื่อคุณ ("ผู้แกะ: …")
>
> ☐ ฉันยืนยันว่าเป็นเพลงที่ฉันแต่งเอง / เพลง public domain / หรือได้รับอนุญาตจากเจ้าของสิทธิ์แล้ว
>
> การเผยแพร่เนื้อเพลงที่มีลิขสิทธิ์โดยไม่ได้รับอนุญาตเป็นความรับผิดชอบของผู้เผยแพร่ ทีมงานจะลบเนื้อหาเมื่อได้รับแจ้งจากเจ้าของสิทธิ์
>
> [ ยืนยันและเผยแพร่ ]   [ ยกเลิก ]
